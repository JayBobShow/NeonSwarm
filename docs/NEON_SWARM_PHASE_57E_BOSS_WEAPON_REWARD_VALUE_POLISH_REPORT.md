# Neon Swarm Phase 57E - Boss Weapon Reward Value Polish Report

## Scope

Phase 57E is a narrow reward-generation-only implementation pass.

The goal is to prevent active Sector 1-3 boss / sector-clear weapon rewards
from feeling weak because they rolled Common or very low-power weapons.

## Implementation

Changed `scripts/NeonSwarm3DGameplayPrototype.gd` only.

Added a constrained sector-clear weapon reward path for `_roll_sector_reward_choices()`.

The constrained path applies only when:

- The reward choice is generated by `_roll_sector_reward_choices()`.
- The generated weapon is the sector-clear / boss-clear weapon reward choice.
- The active sector index is `0`, `1`, or `2`, matching active Sectors 1-3.

The constrained path does not apply to:

- Normal level-up upgrades.
- Armory.
- Forge.
- Equip / Replace / Stash / Scrap route execution.
- Hollow Warden run-complete flow.
- Sector 4 pressure.
- Sector 5 future-locked data.

## Reward Floor

The active Sector 1-3 sector-clear weapon reward floor is:

- Minimum rarity: `Uncommon`
- Minimum power: `1.25`
- Maximum candidate rolls: `10`

The implementation uses the existing weapon reward generator for each candidate.
If a candidate meets both the rarity and power floor, it is accepted immediately.
If all bounded attempts miss the floor, the best candidate from those attempts is
used as a bounded fallback.

## Save And Armory Safety

No save schema changes were made.

No weapon instance fields were added, removed, or migrated.

Existing route behavior is unchanged:

- Equip still writes the accepted generated weapon to an open compatible slot.
- Replace still swaps the accepted generated weapon into the selected slot and
  sends the old weapon to stash.
- Stash still stores the accepted generated weapon.
- Scrap still rejects the accepted generated weapon and grants Neon Dust.

All route validation used isolated temporary user data:

`/tmp/neon_swarm_phase57e_xdg/godot/app_userdata/Neon Swarm`

That temporary user-data directory was removed after validation.

## Known Risks

The fallback remains bounded by design. In an extreme bad roll streak, it can use
the best candidate from the bounded attempts rather than rolling forever. The
10-roll window makes that unlikely in normal play, and validation found no
fallback floor miss.

The weapon reward route flow remains save-adjacent. Future validation that
touches Equip, Replace, Stash, or Scrap must continue using isolated user data.

The 4A `54/54` enemy headroom caveat remains active and unchanged. Do not add
Sector 4 population, event, hazard, or boss pressure until that caveat is
reviewed again.
