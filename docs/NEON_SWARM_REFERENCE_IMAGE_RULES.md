# Neon Swarm Reference Image Rules

Reference images are allowed only to guide original Neon Swarm work. They are
not permission to copy protected designs.

## Reference Categories

User-owned project art:

- Treat as approved project reference when the user identifies it as owned or
  created for Neon Swarm.
- Use it for direct continuity of direction, palette, shape language, UI intent,
  asset quality, and production targets.
- Preserve source files when requested, such as `.blend`, `.glb`, `.kra`,
  `.svg`, or layered working files.

Third-party inspiration:

- Use only for broad direction, vocabulary, quality bars, mood, workflow study,
  material understanding, and visual problem solving.
- Do not copy protected designs, logos, compositions, characters, enemy
  silhouettes, models, textures, kitbash pieces, UI layouts, or distinctive
  arrangements.
- Do not trace, recreate, or closely imitate a single reference.

## Originality Requirement

Final Neon Swarm designs must be original.

Every final asset should have its own:

- Shape identity.
- Color identity.
- Material treatment.
- Silhouette.
- Gameplay purpose.
- Export/source file identity.
- Fit within the approved Neon Swarm visual direction.

## Practical Use

Acceptable use:

- Learning how bevels, panels, glow, glass, emission, UI hierarchy, or hard
  surface materials are commonly handled.
- Comparing whether Neon Swarm art reaches a professional readability and
  finish target.
- Building a mood board of broad qualities from multiple sources.
- Translating user-owned project art into consistent runtime assets.

Unacceptable use:

- Copying a third-party model, texture, logo, character, enemy, arena layout, UI
  screen, icon set, or exact composition.
- Making a close variant of one recognizable reference.
- Using third-party reference as a source asset.
- Importing protected artwork into the project unless the user confirms it is
  licensed for that use.

When in doubt, create a more original design and document the distinction.
